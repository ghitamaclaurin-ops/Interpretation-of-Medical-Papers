# Medical Literature Interpreter

**医学文献解读与报告生成工具** | A Claude Code Custom Skill

面向临床医生的智能文献解读助手，将复杂的医学研究论文转化为结构化、易读的中文报告。

---

## 产品概述

在临床工作繁忙的环境下，医生往往难以抽出时间深入阅读最新医学文献。本工具通过AI智能解析，帮助临床医生在数分钟内获取文献核心信息，支持科室学习、学术交流和文献汇报场景。

### 核心价值

- **节省时间**：无需逐字阅读，快速掌握研究要点
- **专业解读**：基于循证医学标准评估证据等级
- **即用即走**：生成可直接用于汇报和分享的PDF报告
- **中文友好**：专业术语准确翻译，符合国内医学用语习惯

---

## 功能特性

### 1. 智能文献解析
- 自动提取研究设计、方法、主要结果、安全性数据
- 识别研究类型（RCT、队列研究、Meta分析等）
- 结构化呈现PICO要素

### 2. 证据等级评估
- 基于牛津循证医学中心标准
- 自动判定证据等级（Ia-IV级）
- 评估临床推荐强度（A/B/C级）

### 3. 学术反响追踪
- 自动搜索Twitter/学术社区讨论
- 整合PubPeer评论和同行评议
- 仅展示有真实来源的反响信息

### 4. 双格式报告输出

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| **Lancet风格图文摘要** | 单页A3 PDF，信息密度高，视觉化呈现 | 科室晨会、快速分享、社交媒体 |
| **多页详细报告** | 完整分析报告，包含深度解读 | 文献汇报、教学培训、深入学习 |

## 快速开始

### 环境要求

- Python 3.8+
- Claude Code CLI（已安装并登录）

### 安装依赖

```bash
pip install weasyprint reportlab pypdf
```

> **注意**：WeasyPrint 在 Windows 上需要额外安装 GTK3 运行库，详见 [WeasyPrint安装指南](https://doc.courtbouillon.org/weasyprint/stable/first_steps.html)

### 配置技能

1. 将 `medical-literature-interpreter` 文件夹放置在项目根目录
2. 确保 `.claude/settings.local.json` 包含正确的权限配置：

```json
{
  "permissions": {
    "allow": [
      "Skill(medical-literature-interpreter)",
      "WebSearch",
      "Bash(python:*)"
    ]
  }
}
```

### 使用方式

在 Claude Code 中，上传医学文献PDF并输入：

```
请解读这篇文献，生成Lancet风格的图文摘要
```

或

```
请详细分析这篇研究，生成完整报告
```

## 证据等级标准

本工具采用牛津循证医学中心证据等级分类：

| 等级 | 研究类型 | 推荐强度 |
|------|----------|----------|
| **Ia** | 随机对照试验的系统综述/Meta分析 | A级（强推荐） |
| **Ib** | 单个大样本随机对照试验 | A级 |
| **IIa** | 前瞻性队列研究 | B级（中等推荐） |
| **IIb** | 回顾性队列/病例对照研究 | B级 |
| **III** | 病例系列研究 | C级（弱推荐） |
| **IV** | 专家意见/共识 | C级 |

## 常见问题

### Q: 支持哪些类型的医学文献？
A: 支持各类临床研究文献，包括RCT、队列研究、病例对照研究、系统综述、Meta分析等。对于基础研究文献也可解读，但证据评估主要针对临床研究设计。

### Q: 生成的报告可以直接用于学术发表吗？
A: 本工具生成的报告仅供学习和交流使用，不可替代原始文献阅读。报告底部包含免责声明，如用于正式学术场合，请务必核实原文内容。

### Q: 如何保证翻译准确性？
A: 工具内置专业医学术语对照表，涵盖药物名称、统计术语、研究设计术语等。对于新术语，系统会保留英文原文并附中文释义。

### Q: 图文摘要超过一页怎么办？
A: 内置布局优化器会自动检测并调整内容，确保输出为单页PDF。如内容确实过多，会智能精简次要信息。

## 开发说明

### 添加新模板

1. 在 `templates/` 目录创建新的 HTML 模板
2. 在 `scripts/` 目录添加对应的生成脚本
3. 更新 `skill.md` 添加新格式选项

### 扩展证据评估标准

编辑 `references/evidence_levels.md` 文件，可添加其他证据分级体系（如GRADE）。
